Introduccion

El presente documento tiene como objetivo describir el modulo dedicado al gestor de facturación.

    El sistema se desarrollo de acuerdo a las siguientes características generales:
  • Arquitecura APIRest Ful
  • Base de datos relacional desarrollada en MySQL
  • Desarrollado en Python, utilizando Flask
  • Utilización de la extension Thunder Client del IDE Visual Studio Code
  • Desarrollado en HTML, CSS y Javascript

Disclaimer: Existen métodos que han sido desarrollados en el backend sólo con fines academicos. Los mismos no se describiran en esta documentacion, ni se implementaron en el frontend

Version: 1.0.0

Requisitos y autenticacion

Para poder utilizar el sistema, es necesario contar con una cuenta de usuario. Para ello, se debe realizar el siguiente procedimiento:

  1. Acceder a la pagina de inicio de sesion
  2. Seleccionar la opcion "Registrarse"
  3. Completar el formulario con los datos solicitados
  4. Seleccionar la opcion "Registrarse"

Una vez que se haya ingresado al sistema, se podra acceder a las distintas funcionalidades del mismo.

Para poder acceder a las funcionalidades del sistema, es necesario autenticarse. Para ello, se debe realizar el siguiente procedimiento:

  1. Acceder a la pagina de inicio de sesion
  2. Ingresar el correo usuario y la contraseña
  3. Seleccionar la opcion "Iniciar sesion"

Una vez autenticado, se podra acceder a las funcionalidades del sistema.

Para poder cerrar la sesion, Seleccionar la opcion "Cerrar sesion"

Recursos

El sistema cuenta con las siguientes funcionalidades:

  • Gestor de facturacion
  • Gestor de clientes
  • Gestor de productos y servicios
  • Gestor de metricas
  • Gestor de usuarios

Gestor de facturacion

El gestor de facturacion permite realizar las siguientes acciones:

  • Generar una nueva factura
  • Buscar una factura existente
  • Consultar el historial de facturas
Consultas HTTP

El gestor de facturacion permite realizar las siguientes consultas HTTP:

  • GET
    • user/{id_user}/facturas
      • Parametros:
        • id_user: identificador del usuario
      • Respuesta:
        • 200: OK
        • 404: Not found
    • user/{id_user}/facturas/{id}
      • Parametros:
        • id_user: identificador del usuario
        • id: identificador de la factura
      • Respuesta:
        • 200: OK
        • 404: Not found
  • POST
    • user/{id_user}/facturas
      • Parametros:
        • id_user: identificador del usuario
      • Respuesta:
        • 200: OK
        • 404: Not found
